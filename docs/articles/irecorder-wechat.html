我做了个 macOS 小工具：默默记下你打过、复制过的字

iRecorder · 开源 · 本地存储 · 支持中文输入法

你有没有过这种瞬间——

早上在浏览器搜了一串关键词，下午想复用，却怎么也想不起来；

聊天里打了一大段话，发出去前改乱了，想找回上一版；

从网页复制了一段配置，粘到终端后发现粘错了，却不记得原文是什么。

Mac 很擅长「此刻干活」，不太擅长「过会儿回忆」。

于是我做了 iRecorder：一个安静待在菜单栏的小工具，只做一件事——

把你在各 App 里已经打上屏的文字、复制、以及 ⌘V 粘贴，按天写进本机的日志文件。

不截屏、不上云、不弹广告。打开就能用，关掉也干净。

01 · 记的是「结果」，不是按键乱码

很多键鼠记录软件记的是 KeyCode。中文输入时，你会看到一长串拼音和功能键，真正有用的汉字反而埋在里面。

iRecorder 走另一条路：通过系统的辅助功能，读已经出现在输入框里的文本差分。

· 英文：你敲什么，它大致记什么
· 中文：组字过程中的拼音会尽量忽略，只保留确认上屏后的汉字
· 停一会儿（默认约 3 秒）或按 Enter，就落成日志里的一行

复制、粘贴也会记。如果你刚复制完就粘贴同一段，会合并成一行 copy_paste，少一点噪音。

日志长这样（示意）：

2026-07-15T16:12:03+08:00  type  Safari  你好世界
2026-07-15T16:12:10+08:00  copy_paste  Finder→Notes  一段配置文本
2026-07-15T16:12:20+08:00  copy  Safari  only copied

一天一个文件，默认在 ~/Documents/iRecorder/，用记事本、VS Code、甚至 grep 都能查。

02 · 适合谁用？

程序员 / 写作者
回看今天改过哪些句子、搜过哪些报错、从文档里搬过哪些片段。

产品 / 运营
聊天、问卷、飞书里敲过的描述，不用再翻消息记录考古。

轻度强迫症患者
不是要监控别人，只是想给「自己今天输入过什么」留一份可检索的底稿。

如果你需要的是企业级审计、远程监控，它不是那个方向——它更像一本只写给自己的本地输入日记。

03 · 隐私：数据不出你的电脑

· 日志只写在你指定的本地目录
· 不收集、不上传
· 密码框 / 安全输入框里的打字不会记
· 需要你手动授予 macOS「辅助功能」权限——否则读不到别的 App 里的文字

权限看起来吓人，但对这类工具几乎是必选项。你可以随时在菜单里暂停记录，也可以卸载并删掉整个日志文件夹。

另外：高度自绘的界面、游戏、部分 Electron 控件，系统本身就不一定暴露文字，记不全是正常的。

04 · 怎么装？

项目已开源（MIT），需要 macOS 14+，本机有 Swift 工具链。

仓库地址
https://github.com/wizizm/irecorder

终端里三步：

git clone https://github.com/wizizm/irecorder.git
cd irecorder
./scripts/package-app.sh

脚本会编译、打成 App，并装到「应用程序」。第一次打开后：

1. 点菜单栏橙色 iR 图标
2. 去系统设置勾选辅助功能里的 iRecorder
3. 退出 App 再打开一次（macOS 勾完权限，经常要重启进程才生效）

看到当日 log 里出现 session_started，就说明它在干活了。

说明：目前是 ad-hoc 签名，没有走苹果公证。重新安装后，有时要在辅助功能里再勾一下——这是未上架 Mac App Store 的常见取舍。

05 · 写在最后

iRecorder 很小：没有复杂账号体系，没有看板，没有「AI 总结今天你很努力」。

它只是把那些本来会散落在记忆和各个窗口里的字，收成一份按日期排列的文本。

当你某天突然问自己：「早上那串命令到底是什么？」——也许翻一下今天的 .log，答案就在那里。
